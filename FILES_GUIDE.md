## বিস্তারিত ফাইল বর্ণনা

### 1. **app.py** - Flask অ্যাপ্লিকেশন কোর

**উদ্দেশ্য:** সমস্ত সার্ভার-সাইড লজিক পরিচালনা করে

**মূল বৈশিষ্ট্য:**
- মডেল লোডিং: `pickel_model.pkl` থেকে ML মডেল লোড করে
- রুট হ্যান্ডলার:
  - `GET /` - হোম পেজ রেন্ডার করে
  - `POST /predict` - পূর্বাভাস করে
  - `GET /info` - মডেল তথ্য রিটার্ন করে

**গুরুত্বপূর্ণ কার্যকারিতা:**
```python
# মডেল লোড করে
model = pickle.load(open('pickel_model.pkl', 'rb'))

# পূর্বাভাস দেয়
prediction = model.predict(features_array)

# আত্মবিশ্বাস স্তর গণনা করে
probability = model.predict_proba(features_array)
```

**Render এ চলে:**
- Gunicorn দ্বারা পরিচালিত (প্রোডাকশন সার্ভার)
- পোর্ট: 5000 (স্থানীয়) বা Render এর দ্বারা নির্দিষ্ট

### 2. **requirements.txt** - Python ডিপেন্ডেন্সি

**উদ্দেশ্য:** প্রয়োজনীয় সমস্ত Python প্যাকেজ নির্দিষ্ট করে

**বিষয়বস্তু:**
```
Flask==3.0.0              # ওয়েব ফ্রেমওয়ার্ক
Werkzeug==3.0.0           # WSGI ইউটিলিটি
gunicorn==21.2.0          # প্রোডাকশন সার্ভার
numpy==1.24.3             # সংখ্যাগত গণনা
scikit-learn==1.3.0       # মেশিন লার্নিং
```

**কেন প্রয়োজন:**
- Render স্বয়ংক্রিয়ভাবে এটি ব্যবহার করে `pip install -r requirements.txt`
- নিশ্চিত করে সমস্ত ডেভেলপার একই সংস্করণ ব্যবহার করে

### 3. **Procfile** - Render ডিপ্লয়মেন্ট প্রক্রিয়া

**উদ্দেশ্য:** বলে Render কিভাবে অ্যাপ শুরু করতে হবে

**বিষয়বস্তু:**
```
web: gunicorn app:app
```

**অর্থ:**
- `web` = ওয়েব সার্ভিস
- `gunicorn` = প্রোডাকশন সার্ভার
- `app:app` = `app.py` ফাইলের `app` ফ্লাস্ক অবজেক্ট

## স্থানীয় ডায়াবেটিস পূর্বাভাস প্রক্রিয়া

```
ব্যবহারকারী ডেটা প্রবেশ করে
          ↓
HTML ফর্ম যাচাই করে (JavaScript)
          ↓
fetch() API দিয়ে '/predict' এ পোস্ট করে
          ↓
Flask app.py রিসিভ করে
          ↓
ডেটা NumPy অ্যারে তে রূপান্তরিত করে
          ↓
ML মডেল দিয়ে predict() কল করে
          ↓
JSON প্রতিক্রিয়া রিটার্ন করে
          ↓
JavaScript ফলাফল প্রদর্শন করে (HTML/CSS দিয়ে রঙ করা)
```
